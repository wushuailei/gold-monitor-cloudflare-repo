# 定时任务说明

## Cron 配置

```toml
[triggers]
crons = ["*/1 * * * *"]
```

每分钟执行一次。

## 执行流程

```text
1. 数据清理检查（每天 00:00 执行一次）
   └─ 清理 360 天前的历史数据

2. 每日早报检查（每天 09:00 执行一次）
   ├─ 生成 AI 市场分析报告
   ├─ 发送到飞书群
   └─ 保存到 reports 表

3. 市场状态检查
   └─ 如果停盘则跳过后续步骤

4. 拉取金价
   └─ 从数据源获取最新价格

5. 价格异常检测
   └─ 涨跌超过 1000 元则跳过告警

6. 写入数据库
   └─ 插入 prices 表（防重复）

7. 日线汇总
   ├─ 更新 daily_prices 表
   ├─ 记录开盘价（北京时间 9:00）
   └─ 记录收盘价（北京时间 23:55）

8. 告警引擎（仅工作日 9:00-23:00）
   ├─ 计算涨跌幅
   ├─ 判断告警等级
   ├─ 检查节流规则
   └─ 发送飞书告警

9. 目标价检查（仅工作日 9:00-23:00）
   ├─ 检查用户配置的目标价
   ├─ 判断是否触发
   ├─ 更新提醒次数
   └─ 发送飞书提醒

10. 整数关口检查（仅工作日 9:00-23:00）
    ├─ 计算跨越的整数关口
    ├─ 每个关口发送提醒
    └─ 记录到 price_levels 表
```

## 告警时段

只在以下时段发送告警：
- 工作日（周一至周五）
- 北京时间 9:00-23:00

非告警时段仍然会抓取价格和更新日线数据。

## 价格异常检测

为防止数据源异常导致大量告警：
- 涨跌超过 1000 元：跳过所有告警操作
- 关口跨越超过 20 个：跳过关口告警

## 告警引擎规则

### 涨跌幅计算

基于用户配置的对比基准：
- **YESTERDAY**：对比昨日收盘价
- **BUY**：对比未平仓买入价

计算公式：
```
change_percent = (price_now - ref_price) / ref_price * 100
```

### 告警节点

用户可配置三级涨幅节点和三级跌幅节点：
- **rise_1/2/3**：涨幅一/二/三级节点（%）
- **fall_1/2/3**：跌幅一/二/三级节点（%）

### 去重策略

- 一级节点：每天最多提醒 1 次
- 二级节点：每天最多提醒 2 次
- 三级节点：每天最多提醒 3 次

通过 `alerts` 表的去重索引实现。

## 目标价提醒规则

### 触发条件

根据用户配置的比较方式：
- **EQ**：price_now == target_price
- **GTE**：price_now >= target_price
- **LTE**：price_now <= target_price

### 提醒次数

每个目标价最多提醒 3 次：
- 每次触发 alert_count + 1
- 达到 3 次后自动关闭提醒
- 重新开启提醒时重置 alert_count

## 整数关口提醒规则

### 关口计算

步长固定为 10 元：
```
当前关口 = floor(price / 10) * 10
```

### 触发逻辑

每次跨越关口都会提醒：
```
价格从 1105 到 1115：
  - 突破 1110 关口 ✓

价格从 1115 到 1105：
  - 跌破 1110 关口 ✓
```

### 数量限制

单次最多告警 20 个关口（防止异常数据）

## AI 分析触发

### 触发条件

- 告警等级达到 CRIT（三级节点）
- 同一 symbol **30 分钟内无成功报告**

### AI 输入

```json
{
  "symbol": "AU",
  "price_now": 580.32,
  "change_percent": 0.92,
  "base_type": "YESTERDAY",
  "recent_prices": [
    { "ts": 1700000000, "price": 582.1 },
    ...
  ]
}
```

### AI 输出要求

Markdown 格式，包含：
- 可能原因分析
- 短线风险评估
- 操作建议（保守 / 中性 / 激进）

## 数据清理任务

### 执行时机

每天北京时间 00:00-00:05 之间执行一次

### 清理范围

删除 360 天之前的数据：
- prices（价格分钟线）
- daily_prices（日线汇总）
- alerts（告警记录）
- price_levels（关口记录）
- reports（AI 分析报告）
- trades（交易记录）

### 去重机制

使用 KV 存储记录最后清理日期，确保每天只执行一次。

## 每日早报任务

### 执行时机

每天北京时间 09:00-09:05 之间执行一次

### 报告内容

1. **基础数据**
   - 当前金价
   - 昨日收盘价
   - 24小时涨跌幅

2. **AI 分析**
   - 昨日走势回顾
   - 当前市场状态
   - 今日操作建议（保守/中性/激进）

3. **数据来源**
   - 最近30分钟价格序列
   - 昨日收盘价对比
   - 短期涨跌幅计算

### 去重机制

使用 KV 存储记录最后发送日期，确保每天只发送一次。

### 数据保存

报告会保存到 `reports` 表，trigger_type 为 `DAILY`。

## 免费额度控制

### D1 限制
- 读取：500 万行/天
- 写入：10 万行/天

### 当前使用量估算
- prices 写入：~1,440 行/天
- daily_prices 更新：~1,440 行/天
- alerts 写入：< 100 行/天
- price_levels 写入：< 200 行/天
- **总计：~3,300 行/天**（远低于限制）

### 优化措施
- 价格异常检测防止大量写入
- 关口告警数量限制
- 所有查询必须命中索引
- 禁止全表扫描
