# 系统架构说明

**技术栈**：Cloudflare Workers + D1 + Pages  
**采样频率**：1 分钟  
**告警渠道**：飞书群机器人  
**AI 分析**：仅 CRIT 级别触发  
**当前版本**：v1（单品种 AU）

## 核心组件

### 1. Scheduled Worker（定时任务）
- 每分钟执行一次
- 抓取国内金价（AU）
- 计算涨跌幅并触发告警
- 检查目标价触发
- 检查整数关口跨越
- CRIT 级别调用 AI 分析

### 2. API Worker（HTTP 接口）
- 提供价格查询接口
- 交易记录管理
- AI 报告查询
- 告警记录查询
- 关口记录查询
- 用户配置管理
- 持仓信息管理

### 3. Cloudflare Pages（前端）
- 价格走势图表
- 买卖点标注
- AI 分析报告展示
- 目标价管理
- 关口记录展示
- 持仓统计

### 4. D1 数据库
- prices：价格分钟线数据
- daily_prices：日线汇总数据（含开盘/收盘价）
- alerts：告警记录
- price_levels：关口记录
- reports：AI 分析报告
- trades：交易记录
- holdings：持仓信息
- user_targets：用户目标价
- user_configs：用户配置
- global_configs：全局配置

## 数据流向

### 1. 定时任务流
```
Cron → Worker → 金价 API → D1 → 飞书
                    │
                    ├─ prices 表（分钟线）
                    ├─ daily_prices 表（日线）
                    ├─ alerts 表（告警）
                    ├─ price_levels 表（关口）
                    └─ reports 表（AI）
```

### 2. 用户查询流
```
浏览器 → Pages → API Worker → D1 → 返回数据
```

### 3. 用户操作流
```
浏览器 → Pages → API Worker → D1 → 保存数据
                        │
                        └─ holdings 表（持仓）
```

## 告警系统

### 告警类型

| 类型 | 触发条件 | 消息格式 |
|------|----------|----------|
| 涨跌幅节点 | 达到配置的涨跌幅节点 | 🔴 涨幅节点 / 💚 跌幅节点 |
| 目标价触发 | 价格满足目标价条件 | 🎯 目标价触发 |
| 整数关口 | 跨越 10 元整数关口 | 🚀 突破 / 💥 跌破 |
| AI 分析 | CRIT 级别告警时 | 🤖 AI 分析报告 |

### 告警时段
- 工作日（周一至周五）
- 北京时间 9:00-23:00

### 安全机制
- 价格异常检测（涨跌 > 1000 元跳过）
- 关口数量限制（单次最多 20 个）
- 目标价提醒次数限制（每个最多 3 次）

## 免费额度控制

### D1 限制
- 读取：500 万行/天
- 写入：10 万行/天

### 当前使用量
- 每日写入：~3,300 行
- 每日读取：~10,000 行
- **远低于免费额度**

### 优化措施
- 价格异常检测防止大量写入
- 所有查询必须命中索引
- 禁止全表扫描

## v1 的刻意限制

- 只支持 `symbol = 'AU'`（国内金价）
- 不做多周期聚合（5m / 1h）
- 使用 Cloudflare Access 进行访问控制
- 不支持多告警渠道
- 关口步长固定为 10 元
